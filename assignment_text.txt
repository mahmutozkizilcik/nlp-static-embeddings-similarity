Introduction
In this assignment, you will work with static word embeddings and explore how they can be
used to represent words, sentences, and documents in vector space. Word embeddings provide
dense numerical representations of words such that semantically related words tend to appear
closer to each other. These representations have become fundamental tools in Natural Language
Processing, as they enable machines to capture lexical similarity and support downstream tasks
such as similarity analysis, text classification, and information retrieval.
You are required to use pre-trained word embeddings to implement methods for word- and
sentence-level similarity analysis as well as text classification. In the first part of the
assignment, you will examine semantic similarity by retrieving similar words, constructing sen-
tence vectors, comparing sentences in vector space, and analyzing how individual embedding
dimensions contribute to similarity. In the second part, you will use static embedding-based
text representations to build and evaluate a classifier for a text classification task.
The goal of this assignment is to help you understand:
• how static word embeddings represent semantic relationships between words,
• how sentence and document representations can be constructed from word vectors,
• how vector-based similarity can be used to compare words and sentences,
• how static embeddings can support downstream tasks such as text classification,
• and what the strengths and limitations of static embedding-based representations are.

Replace with Similar Words
In this section, given a sentence and a list of indices, the words at the specified indices will be
replaced with their most similar alternatives according to the embedding model.
For this purpose, the function replace_with_similar(sentence, indices) should first be
defined. The sentence parameter will contain a single string, while the indices parameter will
be a list of indices. The indices provided to the function will always be valid, and it is also
guaranteed that each word corresponding to the given indices exists in the vocabulary of the
model.
The sentence will be tokenized based on whitespace. For each token specified by the indices, the
top 5 most similar words along with their similarity scores will be retrieved from the model. For
this purpose, the most_similar() method of the model object should be used. These words
and their scores will be stored in a dictionary, where each original token maps to a list of tuples.
Each tuple contains a similar word and its corresponding similarity score.
Assume that a sentence containing 10 words and a list of two indices are given as parameters,
the structure of the resulting dictionary should be as follows:
{
word_1: [(similar_word_1, score), ..., (similar_word_5, score)],
word_2: [(similar_word_1, score), ..., (similar_word_5, score)]
}
word_1 and word_2 are the words corresponding to the given indices.
Once this dictionary is completed, the original word at each given index will be replaced with the
most similar word having the highest similarity score among the retrieved candidates,
thereby creating a new sentence.
The replace_with_similar() function should return both the dictionary and the new sen-
tence as follows:
(new_sentence, similar_words_dict)
Some usage examples are provided below:
sentence = "I love AIN442 and BBM497 courses"
indices = [1, 5]
new_sentence, most_similar_dict = replace_with_similar(sentence, indices)
print(most_similar_dict.keys(), end="\n\n")
print(most_similar_dict["love"], end="\n\n")
print(most_similar_dict["courses"], end="\n\n")
print(new_sentence)

Find Similar Sentences
The purpose of this section is to identify and list the sentences that are most similar to a given
query using cosine similarity of embedding-based sentence representations. You will have two
different function definitions. These are sentence_vector(sentence) and
most_similar_sentences(file_path, query).
2.2.1
sentence_vector(sentence)
The sentence_vector() function will take only a sentence string and will store the embedding
vectors of each word in the string in a dictionary. This dictionary should look like the following:
{
word_1: [vector with 300 dimensions],
word_2: [vector with 300 dimensions],
...
}
If the word is not found in the model, then a zero vector of the correct dimensionality should
be added to the dictionary.
In order to obtain the sentence vector, the vectors of each word in the dictionary will be summed
up and divided by the number of words. In other words, the mean of each word vectors will be
taken, resulting in a single 300-dimensional sentence vector representation.
The sentence_vector() function should return both the dictionary of each word’s vectors
and the calculated sentence vector as follows:
(vector_dict, sentence_vec)
Here are some examples of how to use the sentence_vector() function:
vector_dict, sentence_vec = sentence_vector("This is a test sentence")
print(vector_dict.keys(), end="\n\n")
print(vector_dict["This"][:5], end="\n\n")
print(vector_dict["a"][:5], end="\n\n")
print(len(vector_dict["test"]))

most_similar_sentences(file_path, query)
The most_similar_sentences() function should take a file path and a query sentence as pa-
rameters. The query sentence will be a string, and the file path will point to a text file containing
different sentences, where each line should be treated as a separate sentence.
First, read the file and treat each line as a separate sentence. Then, using the sentence_vector()
function defined in the previous subsection, generate sentence vectors for both the query and all
sentences in the file. Each sentence should be paired with the query, and the cosine similarity be-

For each sentence, create a tuple that contains the sentence itself and its cosine similarity

(sentence2, cosine_similarity),

Analyze Dimension Contributions
In this section, you will analyze which embedding dimensions contribute most strongly to the
similarity between two words. Rather than treating similarity as a single cosine similarity score,
you will examine how individual dimensions of the embedding vectors reinforce or weaken the
relationship between word pairs.
For this purpose, the function analyze_dimension_contributions(word1, word2, top_k=10)
should be defined. This function will take two words and an integer top_k as parameters. You
may assume that both words exist in the vocabulary of the model.
Let the embedding vectors of word1 and word2 be denoted by u and v. For each dimension i,
compute a contribution score as:
ui · vi
where ui and vi are the values of the two word vectors at dimension i.
After computing these contribution scores for all dimensions, sort them in descending order
according to their absolute values, and return the top k dimensions. The return value should
be a list of tuples in the following format:
[
(dimension_index, contribution_value, word1_value, word2_value),
...
]
Here, dimension_index is the index of the embedding dimension, contribution_value is the
product ui ·vi, and word1_value and word2_value are the corresponding values of the two word
vectors in that dimension.
For example, a function call may look as follows:
results = analyze_dimension_contributions("good", "great", top_k=5)
for item in results:
print(item)
In addition, you are provided with a file named word_pairs.txt, where each line contains a
pair of words separated by whitespace.
word1 word2
You are expected to apply your function to the given word pairs and examine how dimension-
level contribution patterns differ across semantically similar, contrastive, and unrelated pairs.
This section is intended to help you interpret similarity not only at the vector level, but also at
the level of individual embedding dimensions, allowing you to better understand how different
parts of an embedding jointly contribute to semantic relatedness.

You will work on a binary text classification task. Each text instance will be converted into

These document vectors will then be used to train and evaluate a classifier.
The goal of this part is to help you understand how static word embeddings can support text
classification, how word-level representations can be aggregated into document-level features,
and what the strengths and limitations of such representations are in supervised learning tasks.

Construct Document Vectors
The purpose of this section is to represent each review using a single fixed-size vector derived
from static word embeddings.
For this purpose, the function document_vector(text) should be defined. The text param-
eter will contain a single review string. The review should be tokenized based on whitespace.
For each token, retrieve its embedding vector from the pre-trained word2vec-google-news-300
model.
If a word is not found in the model, it should be ignored in the document representation.
If none of the words in a review exist in the model vocabulary, the function should return a zero
vector of the correct dimensionality.
In order to obtain the document vector, compute the mean of all valid word vectors found
in the review.
The document_vector() function should return a single vector representation of the input
review.
For example, a function call may look as follows:
vec = document_vector("This movie was great and very enjoyable")
print(len(vec))
